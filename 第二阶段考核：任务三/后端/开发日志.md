运行逻辑：
全局异常处理
        一、注册流程
                1. 前端发 POST 请求到 /api/user/register，带上用户名、邮箱、手机号、密码、确认密码
                2.Controller 接收请求，转成 RegisterReq 对象
                3.Controller 调用 Service 的 register 方法
                4.Service 校验两次密码是否一致
                5.Service 查询数据库，校验用户名是否已存在
                6.Service 查询数据库，校验邮箱是否已存在
                7.Service 创建 User 对象，用 BCrypt 加密密码
                8.Service 调用 Mapper 的 insert 方法存入数据库
                9.Service 用 JWT 生成 token
                10.Controller 把 token 返回给前端
        二、登录流程
                1. 前端发 POST 请求到 /api/user/login，带上账号和密码
                2.Controller 接收请求，转成 LoginReq 对象
                3.Controller 调用 Service 的 login 方法
                4.Service 根据邮箱或手机号查询用户
                5. 查不到则抛异常用户不存在
                6. 用 BCrypt 比对密码是否正确
                7. 密码错误则抛异常密码错误
                8.Service 用 JWT 生成 token
                9.Controller 把 token 返回给前端
        三、获取个人信息流程
                1. 前端发 GET 请求到 /api/user/info，Header 带上 Authorization Bearer token
                2.AuthInterceptor 拦截请求，解析并校验 token
                3.token 无效返回 401，有效则放行
                4.Controller 从 Header 取出 token，解析出 userId
                5.Controller 调用 Service 的 getUserInfo 方法
                6.Service 根据 userId 查询数据库
                7.Controller 把用户信息返回给前端
